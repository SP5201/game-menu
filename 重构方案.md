# QDesktop 重构方案

> 目标：**让结构清晰、代码简洁**。
> 编译环境：Delphi XE2（`D:\办公软件\Delphi XE2`），验证命令 `编译测试.bat`（msbuild Win32/Win64）。

---

## 一、现状诊断（基于实读代码，非推测）

### 1.1 核心病灶：`窗口\UI_MainWindow.pas` 是 God Class
- **2184 行**，单一 `TMainFormUI` 类承载 **6 个互不相关的关注点**：分类列表 / 过滤工具栏 / 全盘搜索编排 / 拖放增删 / 排序 / 窗口几何。
- 22 个 `class var`、约 50 个 `class procedure/function static`。

### 1.2 利好：`TMainFormUI` 是叶子节点，零外部耦合
全仓 grep 确认：**除 `UI_MainWindow.pas` 自身外，没有任何 `.pas` 单元 `uses UI_MainWindow`**。唯一外部消费者是 `QDesktop.dpr`（仅调 `TMainFormUI.LoadLayout`）。
→ **拆分该文件零外部风险**，无需改任何调用方。

### 1.3 次要问题：消息桥模式重复 4 处
`New(payload)→Post→WndProc分发→Dispose` + `Drain/PeekMessageW` 循环，重复在 DiskSearch / ShellIconLoader / EverythingIndex / UI_MainWindowStat。
- **`WM_QD_INDEX_HIT_CHANGES`(116) 在 DiskSearch.pas:12 与 EverythingIndex.pas:375 重复定义** —— 隐患。
- SafeLog 用 `WPARAM(entryId)` 轻量传参（无 heap），与其余不同，不纳入统一。

## 二、分阶段路线图

| 阶段 | 内容 | 风险 | 收益 |
|------|------|------|------|
| **一** | 拆分 `UI_MainWindow.pas` 为 6 个职责单元 | 低（叶子节点） | 立竿见影，文件降到 ~350 行 |
| **二** | 统一消息桥 + 消除重复常量 | 中（改调用点） | 消除 4 处重复、修隐患 |
| **三** | 状态从 `class var` 迁移到实例对象 + Controller 层 | 高（近重写） | 彻底解耦 |

## 三、阶段一：拆分 `UI_MainWindow.pas`

### 3.1 拆分原则
- **纯文本搬运**：方法体、签名、常量、实现一字不改；唯一允许的改写是补 `uses`。
- **class var 字段定义全部留在 `UI_MainWindow.pas` 的类声明里**（Delphi 同一类的 class var 只能声明一处），其余单元通过 `TMainFormUI.CXxx` 访问。
- 同一类的不同方法实现可分布在多个单元（XE2 支持）。
- 每拆一个单元 → 单独提交 → 跑 `编译测试.bat` → 通过再继续。
- 新单元在 `QDesktop.dproj` 各 `<None Include>` 补一行；新单元只被 `UI_MainWindow` 内部 uses，无需改 `.dpr`。

### 3.2 目标目录结构（全部新建于 `窗口\` 下）
```
窗口\
  UI_MainWindow.pas              # 收缩为 ~350 行：窗体壳 + Init/Destroy + LoadLayout + WndProc 分发
  UI_MainCategory.pas   (新)     # 分类列表：加载/选中/增删改菜单
  UI_MainListFilter.pas (新)     # 文件类型过滤工具栏（9 按钮 + 计数 + ExtId 映射）
  UI_MainDiskSearch.pas (新)     # 全盘搜索编排：请求/结果应用/后台排序/缓存/索引状态
  UI_MainListActions.pas(新)     # 拖放/添加文件夹/右键菜单(打开/编辑/删除)/批量插入
  UI_MainListSort.pas   (新)     # 排序按钮/菜单/ApplyListSort
  UI_MainWindowGeometry.pas(新)  # 窗口几何 + 默认样式 + 布局辅助
```

### 3.3 成员 → 单元 映射表（精确到方法）

**UI_MainCategory.pas**：`LoadListBoxFromStore` `ReloadCategoriesAndSelectGroup` `SyncCategoriesForDrop` `OnCategorySelect` `OnCategoryMenuSelect` `SelectSearchCategoryItem` `ReloadListForCategoryIndex` `LoadListViewFromStore`；自由过程 `NormalizeCategoryIconFile`。常量 `ID_LISTBOX_CATEGORY_*` 定义在 `UI_ListBox.pas:9-13`，新单元 uses UI_ListBox 即可。

**UI_MainListFilter.pas**：`SetListFilterSource` `SyncListFilterSourceFromView` `ApplyListFilterToView` `ApplySearchHitsToViewPreserveScroll` `UpdateListFilterButtonStyles` `SelectListFilter` `RemoveListFilterSourceItem` `UpdateListFilterSourceItem` `OnListFilterButtonClick` `RefreshListFilterButtonCounts`；文件级过程 `ListFilterExtInSet/EnsureExtIdMap/ExtFromPath/AssignSourceItem/HitKindIndex/ItemKindIndex/SetListFilterButtonCount/SetListFilterButtonVisible/AdjustListFilterToolbarLayout`；常量 `cListFilter*`（含 9 个 PChar 数组 + 扩展名集合，须同单元内保持在一起）；全局变量 `GExtIdToFilterKind` `GExtIdToFilterKindBuiltFor`。

**UI_MainDiskSearch.pas**：`SearchUiDebug` `RequestDiskSearch` `ApplyDiskSearchResults` `StopSearchHitSortThread` `SaveSearchResultCache` `ApplySearchViewState` `ScheduleSearchHitSort` `ApplySearchSortDone` `HandleDiskSearchError` `ShowDiskSearchError` `BridgeDiskSearchResult` `BridgeDiskSearchError` `DrainPendingDiskSearchMessages` `ApplyIndexHitChanges` `RefreshSearchIndexStatusLabel` `StartSearchIndexStatusTimer` `OnSearchDebounceTimer` `ScheduleDebouncedDiskSearch` `PerformDiskSearchFromBox` `OnMainSearchButtonClick` `ApplyListIconLoadResult`；内嵌类 `TSearchHitSortThread`；常量 `cMainSearchDebounceTimerId/Ms` `cMainIndexStatusTimerId/Ms`。

**UI_MainListActions.pas**：`OnContentMenuSelect` `OnWndDropFiles` `InsertBatchItemsToUI` `LogGroupDbOp`；自由过程 `ListViewItemToLibraryItem` `LibraryItemToListViewItem` `InvalidateListViewItemIcon` `MainWindowListViewPrepareContextMenu` `MainWindowListViewItemActivate` `MainWindowResolveListViewMenuFilePath` `MainWindowListViewItemWidthChanged`；辅助 `MainWindowIsSearchListContext`；全局变量 `GListViewContextItemIndex` `GListViewContextFilePath`；常量 `ID_LISTVIEW_MENU_*`。

**UI_MainListSort.pas**：`ApplyListSort` `OnListSortButtonClick` `OnListSortMenuSelect`；常量 `ID_MAIN_LIST_SORT_*`。

**UI_MainWindowGeometry.pas**：自由过程 `ApplyDefaultStyles` `ClampWindowBoundsToWorkArea` `RestoreMainWindowBounds` `SaveMainWindowBounds` `ApplyListViewLayoutFromConfig` `AdjustMainSearchAreaLayout`。

**UI_MainWindow.pas（壳）**：保留类声明（22 个 class var 定义）、`Init` `Destroy` `LoadLayout` `OnWinProc` `OnListLayoutButtonClick` `OnSafeLogButtonClick`；implementation uses 全部 6 个新单元。

### 3.4 uses 关系
- `UI_MainWindow.pas` implementation uses → 6 个新单元。
- 各新单元 implementation uses → `UI_MainWindow`（访问 class var）+ 各自依赖。
- 避免循环：class var 集中在 UI_MainWindow interface 类声明，新单元只读不重定义。

### 3.5 阶段一执行顺序（每步一 commit + 编译测试.bat）
```
0. git checkout -b refactor/split-main-window
1. 拆 UI_MainWindowGeometry.pas（最独立，无 class var 依赖）→ 编译测试.bat
2. 拆 UI_MainListSort.pas                           → 编译测试.bat
3. 拆 UI_MainListFilter.pas                         → 编译测试.bat
4. 拆 UI_MainDiskSearch.pas                         → 编译测试.bat
5. 拆 UI_MainCategory.pas（uses UI_ListBox 取分类菜单 ID）→ 编译测试.bat
6. 拆 UI_MainListActions.pas                        → 编译测试.bat
7. 更新 QDesktop.dproj 补 6 个 <None Include>      → 编译测试.bat
8. 全功能手动回归 → 合并
```
任一步失败即 `git reset --hard` 回退。

## 四、阶段二：统一消息桥（阶段一稳定后）

### 4.1 新增 `功能\并发\UiThreadMessage.pas`
- `UiMsgPost(AMainWindow, AMsg, APayload)` —— 内含 XC_PostMessage→PostMessageW→Dispose 三级回退。
- `UiMsgDrain(AMainWindow, AMin, AMax, AHandler)` —— PeekMessageW 循环。

### 4.2 改造点
| 模块 | 现状 | 改为 |
|------|------|------|
| DiskSearch `PostSearchResult/Error` | 自写三级回退 | UiMsgPost |
| ShellIconLoader `PostIconResult` | 自写三级回退 | UiMsgPost |
| EverythingIndex `PostIndexHitChanges` | 单层 PostMessageW | UiMsgPost |
| UI_MainWindowStat `TryPostStatMessage` | 单层 XC_PostMessage | UiMsgPost |
| UI_MainDiskSearch `DrainPendingDiskSearchMessages` | 自写 PeekMessageW | UiMsgDrain |
| UI_MainWindowStat `DrainPendingStatMessages` | 自写 PeekMessageW | UiMsgDrain |

### 4.3 消除重复常量
新增 `功能\并发\UiMessageIds.pas` 集中所有 `WM_QD_*` 常量；删除 DiskSearch.pas:12 与 EverythingIndex.pas:375 对 `WM_QD_INDEX_HIT_CHANGES` 的重复定义，改为 uses 共享常量。

## 五、阶段三：实例化状态 + Controller 层（高风险，可选）

把 22 个 class var 迁移到实例字段；抽取 `TCategoryController`/`TSearchController`/`TListFilterController`；事件从 `class procedure static + @TMainFormUI.OnX` 改为实例方法。
**风险**：XCGUI 大量 `@TMainFormUI.OnXxx` 取类方法地址，改实例方法需逐一改 RegEvent；近重写。仅在你愿逐功能回归时推进。
